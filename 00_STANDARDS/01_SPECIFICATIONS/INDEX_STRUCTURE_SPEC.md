# UNIVERSE ENGINE — INDEX STRUCTURE SPEC
## Спецификация структуры и иерархии INDEX-файлов
## Уровень: L1 · Mandatory

---

## 0. НАЗНАЧЕНИЕ ДОКУМЕНТА

Этот документ определяет:
- что такое INDEX в системе Universe Engine
- какие типы INDEX-файлов существуют
- их иерархию и зоны ответственности
- правила добавления и изменения информации в INDEX

INDEX — это логическая карта системы,
а не файловое хранилище.

---

## 1. ЧТО ТАКОЕ INDEX

INDEX — это структурированный навигационный документ,
который описывает:

- существующие элементы системы
- связи между ними
- роли и участие в процессах
- текущее состояние и статус

INDEX не содержит самих данных,
а указывает, **где и как они используются**.

---

## 2. ТИПЫ INDEX-ФАЙЛОВ

В системе существуют три типа INDEX:

### 2.1 Главный INDEX
`INDEX__UNIVERSE_ENGINE.md`

Роль:
- глобальная карта всей системы
- реестр всех UID
- описание всех доступных возможностей
- источник истины для всех остальных INDEX

---

### 2.2 Доменные INDEX
Примеры:
- `INDEX__NARRATIVE.md`
- `INDEX__VISUAL.md`
- `INDEX__AUDIO.md`

Роль:
- применение глобальных правил к конкретной области
- детализация процессов и сущностей домена
- не вводят новые правила без регистрации

---

### 2.3 Проектные INDEX
Примеры:
- `INDEX__PROJECT_ALPHA.md`
- `INDEX__FILM_MARS.md`

Роль:
- описание конкретного проекта
- фиксация входных данных
- использование сущностей и моделей

---

## 3. ИЕРАРХИЯ АВТОРИТЕТА

Иерархия строго фиксирована:

SYSTEM CANON  
↓  
INDEX__UNIVERSE_ENGINE  
↓  
Domain INDEX  
↓  
Project INDEX

Правила:
- нижестоящий INDEX не может противоречить вышестоящему
- конфликт разрешается только на более высоком уровне
- локальные INDEX не переопределяют канон

---

## 4. ОБЯЗАТЕЛЬНЫЕ РАЗДЕЛЫ INDEX

Каждый INDEX обязан содержать:

- PURPOSE — назначение
- SCOPE — границы действия
- USED_ENTITIES — используемые сущности (UID)
- USED_MODELS — используемые модели (UID)
- FILE_MAP — ссылки на реальные файлы
- EVENTS / STATES — события или состояния
- STATUS — актуальность INDEX

---

## 5. РЕГИСТРАЦИЯ ЭЛЕМЕНТОВ

Любой новый элемент:
- сущность
- модель
- база
- процесс

обязан:
1. получить UID
2. быть зарегистрирован в главном INDEX
3. только после этого использоваться в доменных или проектных INDEX

---

## 6. СТАТУСЫ В INDEX

Элемент в INDEX имеет статус:

- planned — запланирован
- active — используется
- deprecated — выведен из использования

Статус обязателен для всех значимых элементов.

---

## 7. ЗАПРЕТЫ (СТРОГО)

Запрещено:
- хранить данные внутри INDEX
- дублировать один и тот же элемент в разных INDEX
- вводить локальные правила без регистрации
- использовать элементы без UID

---

## 8. INDEX И ЧАТЫ

Чаты:
- читают INDEX
- не изменяют INDEX напрямую
- работают с UID и ссылками

Любое изменение формулируется как:
- предложение
- анализ
- фиксация в INDEX

---
