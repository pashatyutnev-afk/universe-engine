## 0) WHAT IS KB
KB (Knowledge Base) — каноническое хранилище знаний о мире/персонажах/объектах/событиях/правилах.
KB — это **не проект и не производство**, а “истина мира”.

---

## 1) CORE LAW
- Любая KB-сущность обязана иметь UID.
- Любая KB-сущность обязана быть зарегистрирована в KB Global Registry.
- Если сущности нет в реестре — она считается NON-CANON (ignored).

---

## 2) HOW TO USE
1) Находишь нужный тип (CHR/LOC/OBJ/EVT/…).
2) Создаёшь паспорт сущности по шаблону (минимальные поля).
3) Регистрируешь сущность в `02__INDEX__KB_GLOBAL_REGISTRY.md`.
4) Прописываешь связи (XREF/DEPENDS_ON) строго по правилам.
5) Только после этого сущность считается “в системе”.

---

## 3) WHAT KB IS NOT
KB не хранит:
- черновые “рабочие” версии сцен/монтажа/промптов проекта
- выходные файлы (видео/аудио/финальные сцены)
Это живёт в PRJ/OUT/ASSETS.
