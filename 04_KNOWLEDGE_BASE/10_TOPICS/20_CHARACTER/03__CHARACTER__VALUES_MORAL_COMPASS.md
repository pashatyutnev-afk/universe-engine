## 1) PURPOSE
Сделать ценности персонажа **операциональными**:
- не “он добрый”, а “какую линию он не перейдёт”,
- где он готов компромиссить,
- что для него важнее в конфликте ценностей,
- как это проверяется в сценах.

---

## 2) WHEN TO USE
- Персонаж резко меняет моральную позицию без давления.
- Сцены “морального выбора” не работают (нет ставок или линии).
- Нужно сделать конфликты ценностей (value conflict) двигателем действий.

## 3) WHEN NOT TO USE
- Ты определяешь WANT/NEED/FEAR (другой topic).
- Ты пишешь клиническую психологию (не требуется).
- Ты создаёшь чисто “характер-портрет” без сцен (лучше начать с мотивации).

---

## 4) INPUTS (MINIMUM)
- Контекст истории (что на кону).
- Мотивация персонажа (WANT/NEED/FEAR желательно известны).
- Оппозиция/давление (кто/что заставляет нарушать ценности).

## 5) OUTPUTS (MINIMUM)
- Value Card: 3 ценности + 2 non-negotiables + compromise ladder.
- 3 boundary tests (сценарии давления).
- PASS/REWORK/FAIL по согласованности ценностей.

---

## 6) DEFINITIONS (STRICT)
### Value (ценность)
То, что персонаж защищает и предпочитает, даже если это дорого.

### Non-negotiable (линия)
То, что персонаж **не сделает**, даже если это выгодно (пока линия не сломана драматически и с ценой).

### Compromise ladder (лестница компромисса)
Порядок “что он уступит первым, а что последним”.

### Value conflict
Ситуация, где две ценности или ценность и цель конфликтуют — нужен выбор.

---

## 7) PROCEDURE (VALUE CARD + LADDER)
### Step 1 — Pick 3 VALUES (verbs, not adjectives)
Формулируй как действие:
- “защищать слабых”
- “держать слово”
- “быть свободным”
Запрещено: “добрый”, “смелый” без действия.

### Step 2 — Define 2 NON-NEGOTIABLES (hard lines)
Формат:
- “Я НЕ ___ даже если ___.”
Примеры:
- “Я не предам друга даже если меня арестуют.”
- “Я не убью безоружного даже если это самый лёгкий выход.”

### Step 3 — Build COMPROMISE LADDER (5 rungs)
Запиши 5 уровней уступки от лёгкого к невозможному:
- R1: уступит легко (мелочь/комфорт)
- R2: уступит с раздражением (репутация/время)
- R3: уступит с ценой (деньги/ресурс/отношение)
- R4: почти не уступит (мораль/идентичность)
- R5: не уступит (non-negotiable)

### Step 4 — Define VALUE PRIORITY RULE
Если ценности конфликтуют, что важнее?
Формат:
- “Если A конфликтует с B, я выбираю A, пока ___ (условие).”

### Step 5 — Boundary tests (3 tests)
Придумай 3 давления:
- TEST 1: быстрый выигрыш требует маленькой грязи
- TEST 2: спасение себя требует предательства
- TEST 3: достижение цели требует нарушения non-negotiable
Для каждого:
- выбор
- цена
- последствия

### Step 6 — Scene usability check
Проверь:
- можно ли по Value Card предсказать выбор героя в моральной сцене?
Если нет — перепиши ценности в действия и уточни линии.

---

## 8) QUICK CHECKLIST (PASS/FAIL)
- [ ] 3 values сформулированы действиями
- [ ] 2 non-negotiables прописаны как “не сделаю даже если…”
- [ ] есть compromise ladder (5 уровней)
- [ ] есть правило приоритета при конфликте ценностей
- [ ] есть 3 boundary tests с ценой/последствием
- [ ] выбор героя предсказуем из Value Card

PASS IF:
- все пункты true

REWORK IF:
- ценности есть, но не связаны с линиями/лестницей

FAIL IF:
- ценности абстрактные и не влияют на решения

---

## 9) EXAMPLES (MANDATORY)
### 9.1 PASS EXAMPLE
VALUES:
1) “держать слово”
2) “защищать слабых”
3) “не лгать близким”
NON-NEGOTIABLES:
- “Я не предам друга даже если меня арестуют.”
- “Я не убью безоружного даже если это спасёт меня.”
COMPROMISE LADDER:
- R1: пожертвовать комфортом
- R2: пожертвовать временем/репутацией
- R3: пожертвовать деньгами/ресурсом
- R4: солгать врагу (но не близким)
- R5: не предать друга (линия)
BOUNDARY TEST:
- предлагают свободу за донос на друга → отказывается, платит ценой ареста.
WHY PASS:
- линии и лестница делают выбор предсказуемым и драматичным.

### 9.2 FAIL EXAMPLE
VALUES:
- “быть хорошим”
NON-NEGOTIABLES:
- нет
WHY FAIL:
- нет действий, нет линий, нет проверок → любые решения выглядят случайными.

### 9.3 EDGE EXAMPLE (value break)
NON-NEGOTIABLE:
- “не предам друга”
В истории:
- герой предаёт друга под экстремальным давлением.
Требование:
- это должно быть turning point с ценой и необратимостью (иначе будет random shift).
WHY EDGE:
- ломать линию можно, но только как крупный поворот с последствиями.

---

## 10) COMMON FAILURE MODES
- Failure: values = прилагательные
  Fix: переписать в глаголы (действия).
- Failure: нет лестницы
  Fix: добавить 5 уровней уступок.
- Failure: “линия ломается без цены”
  Fix: оформить как turning point с cost/irreversibility.
- Failure: ценности не проверяются сценами
  Fix: сделать 3 boundary tests.
