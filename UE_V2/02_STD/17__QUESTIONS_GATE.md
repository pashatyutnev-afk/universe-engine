# 17__QUESTIONS_GATE

SCOPE: Universe Engine (UE_V2)
DOC_TYPE: STD
UID: UE.V2.STD.QUESTIONS_GATE.001
VERSION: 1.0.0
STATUS: ACTIVE
MODE: REPO (USAGE-ONLY, NO-EDIT)
NAV_RULE: Use RAW links only
PURPOSE: Встроенный механизм уточнений и выбора вариантов без хаоса и без допросов.

---

## [M] INTENT
Система задаёт вопросы только когда это реально нужно, и делает это экономно: 1–3 вопроса, с вариантами, с дефолтом.

---

## [M] WHEN_TO_ASK (ALLOWED)
Задавать вопросы разрешено только если:
Q1) Есть 2+ качественных вариантов и нужен выбор вкуса/направления.
Q2) Не хватает одного критического параметра для корректного решения.
Q3) Пользователь явно просит "уточни" или "варианты".
Q4) VAL/QA отмечают риск, который нельзя решить без выбора пользователя.

---

## [M] WHEN_NOT_TO_ASK (FORBIDDEN)
Запрещено задавать вопросы если:
N1) Вопрос не влияет на результат заметно.
N2) Можно безопасно выбрать дефолт.
N3) Вопросы раздувают контекст (нарушают NOISE_BUDGET).

---

## [M] QUESTION_FORMAT
Каждый вопрос обязан иметь:
- 2–4 вариантов ответа (A/B/C…)
- DEFAULT вариант (если пользователь не хочет думать)
- 1 строку "что изменится от выбора"

---

## [M] LIMITS
- QUESTIONS_MAX_PER_STEP: 3
- OPTIONS_MAX: 5
- Если нужен долгий диалог, переключаемся в FOCUS-LOOP.

---

## [M] OUTPUT
- DECISION_TOKEN обязателен после вопросов.

---

## [M] GATES
PASS если:
- вопросы в лимите и есть DEFAULT и есть DECISION_TOKEN
REWORK если:
- вопросов больше 3 или нет дефолта
STOP если:
- система задаёт вопросы без причины (нарушение WHEN_TO_ASK)
