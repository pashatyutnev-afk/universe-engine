## [M] PURPOSE
PIPELINE_CONTRACT — роутер governance-реалма QA.
Определяет порядок governance-правил QA и как собирать единое решение.
RAW не хранит: работает через KEY и INDEX_MANIFEST.

## [M] HARD_RULES
- RAW запрещён внутри CONTRACT.
- Все ссылки: только KEYS.
- MODE REPO: никаких правок репозитория.
- Выход: QA_GVN_PLAN (KEYS) + PRIORITY_ORDER + CONFLICT_RULES.

## [M] REQUIRED_KEYS
- INDEX_MANIFEST
- PIPELINE_CONTRACT
- QA.GVN.ISSUE_LIFECYCLE
- QA.GVN.CREATE_FLOW
- QA.GVN.NATURALNESS_GATE

## [M] PRIORITY_ORDER (highest first)
1) QA.GVN.ISSUE_LIFECYCLE
2) QA.GVN.CREATE_FLOW
3) QA.GVN.NATURALNESS_GATE

## [M] CONFLICT_RULES
- FAIL dominates everything (stop, return REQUIRED_FIXES + ISSUES).
- ASK dominates PASS/WARN (request minimal missing inputs).
- WARN returns REQUIRED_FIXES but may continue.
- PASS only when no higher-severity result exists.

## [M] DEFAULT_PLAN (keys)
- ALWAYS:
  - QA.GVN.ISSUE_LIFECYCLE
  - QA.GVN.CREATE_FLOW
  - QA.GVN.NATURALNESS_GATE

## [M] STEP-RUN
- STEP: S0
  GOAL: Return governance plan keys
  INPUTS: [TASK_TEXT?, ARTIFACT_HINTS?]
  TARGETS: [INDEX_MANIFEST]
  ACTIONS:
    - Output QA_GVN_PLAN = DEFAULT_PLAN
  OUTPUTS: [QA_GVN_PLAN]
  CHECKS: [REQUIRED_KEYS_PRESENT]
  FAIL: UE.FAIL.MISSING_KEY
  NEXT: "го"

- STEP: S1
  GOAL: Aggregate governance decisions
  INPUTS: [QA_GVN_PLAN, CONTEXT_TOKENS?]
  TARGETS: []
  ACTIONS:
    - Apply entities in PRIORITY_ORDER
    - Aggregate using CONFLICT_RULES
  OUTPUTS: [QA_GVN_DECISION, REQUIRED_FIXES, ISSUES?]
  CHECKS: [DECISION_PRODUCED]
  FAIL: UE.FAIL.GATE_FAIL
  NEXT: "го"

## [M] FAIL_CODES
- UE.FAIL.MISSING_KEY
- UE.FAIL.GATE_FAIL
- UE.FAIL.RULE_VIOLATION
- UE.FAIL.INPUT_ABSENT
