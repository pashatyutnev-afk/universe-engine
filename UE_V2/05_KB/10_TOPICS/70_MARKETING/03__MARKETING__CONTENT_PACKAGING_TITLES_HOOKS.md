## 1) PURPOSE
Упаковка = сделать так, чтобы релиз:
- кликали (title),
- досматривали первые секунды (hook line),
- понимали ценность (description),
- находили (tags/hashtags),
и чтобы всё было едино с позиционированием.

---

## 2) WHEN TO USE
- Публикация трека/клипа/shorts.
- Заголовки “никакие”, CTR низкий.
- Описание без смысла или слишком длинное.
- Нужны разные упаковки под разные сегменты.

## 3) WHEN NOT TO USE
- Внутренний архив без внешней публикации (но часть полей всё равно нужна для каталога).

---

## 4) INPUTS (MINIMUM)
- Positioning Card (promise + tone + forbidden drift).
- Audience segments/personas (optional but recommended).
- Track hook line (1 строка) + 2–3 ключевых маркера стиля.
- Platform targets: YouTube / Shorts / TikTok / etc.

## 5) OUTPUTS (MINIMUM)
- 3 title variants (platform-appropriate).
- 2 hook lines (1 для short, 1 для full).
- 1 short description + 1 long description.
- Hashtag set (5–12) + keyword set (5–12).
- PASS/REWORK/FAIL.

---

## 6) PRINCIPLES (STRICT)
- P1: Title must be specific (style or promise, not “cool track”).
- P2: Hook line must fit first 2–3 seconds (shorts).
- P3: Description must state value + vibe + one proof signal.
- P4: No spam tags; tags support discovery, not noise.
- P5: Packaging must match tone rules (no drift).

---

## 7) TITLE RECIPES (COPY & ADAPT)
### T1 — Classic (safe)
<ARTIST> — <TRACK> (Official Audio) [vX.Y.Z]

### T2 — Style-forward
<TRACK> — <STYLE_FINGERPRINT> (Rap Rock / Industrial Nu Metal)

### T3 — Promise-forward
<TRACK> — <PROMISE LINE> (Official)

### T4 — Shorts variant
<TRACK> — <HOOK PHRASE> (Short)

### T5 — Clip/UGC
<TRACK> — <2-WORD TAG> / <Moment name>

Rules:
- Title ≤ 70 chars preferred (YouTube).
- For shorts: keep it short and readable.

---

## 8) HOOK LINES (TEXT HOOKS FOR POSTS)
Типы:
- H1: Quote (1 строка из припева)
- H2: Challenge (вызов)
- H3: Problem→solution (шум→сигнал)
- H4: Call-response (вопрос→крик)

Templates:
- Quote: "«___»"
- Challenge: "Скажи мой режим — ___"
- Problem→solution: "Сбо́и в голове — но маршрут есть."
- Call-response: "Сейчас — назови мой режим… / СЕЙЧАС!"

Rule:
- 1 hook line = 1 смысл, без воды.

---

## 9) DESCRIPTION TEMPLATES
### 9.1 SHORT DESCRIPTION (1–2 lines)
- 1 line: promise / vibe
- 1 line: style fingerprint (2–4 жанра) + CTA

Template:
<Promise sentence>. <Style fingerprint>.  
Слушай полностью / забери шорт-луп.

### 9.2 LONG DESCRIPTION (5–12 lines)
Обязательные блоки:
- 1: Promise / mood
- 2: Style fingerprint + bpm/feel
- 3: Credits (минимум)
- 4: Rights note (если нужно)
- 5: CTA + hashtags

Template:
1) <Promise / mood>  
2) Style: <genres>. BPM: <x>. Feel: <half-time stomp>.  
3) Credits: Lyrics — __ / Prod — __ / Vocals — __  
4) Rights: <note>  
5) #tags...

---

## 10) HASHTAG / KEYWORD RULES
- HS1: 5–12 hashtags max (shorts)
- HS2: 5–12 keywords (YouTube tags)
- HS3: 60% style, 40% mood/topic
- HS4: include 1–2 brand tags (project/group)
- HS5: no irrelevant high-volume spam

Hashtag buckets:
- Style: #numetal #industrialrock #raprock #electronicmetal
- Mood/Theme: #glitch #signal #noise #reboot
- Format: #shorts #reels (if needed)
- Brand: #<project_tag>

## 12) QA GATES (PASS/FAIL)
PASS IF:
- есть 3 разных title variants
- есть hook line, пригодная для первых 2–3 секунд
- short/long description заполнены
- tags/hashtags в лимитах и по делу
- тон соответствует positioning card

REWORK IF:
- заголовки слишком похожи, или hook line слабый

FAIL IF:
- “для всех”, клише, спам-теги, drift

---

## 13) EXAMPLES (MANDATORY / GENERIC)
Title (style-forward):
- "СБОИ — electronic nu metal / industrial rap rock"
Hook line:
- "Сбо́и в голове — но я вижу маршрут."
Short desc:
- "Шум режет мысль, но сигнал возвращается. 94 bpm half-time stomp."
WHY PASS:
- конкретно, узнаваемо, под платформу.
