## 1) PURPOSE
Кульминация и развязка — это “оплата” всех обещаний истории:
- решающий выбор (decisive choice),
- пик цены (peak cost),
- сбор всех setup/payoff,
- новый устойчивый state (after).

---

## 2) WHEN TO USE
- Финалы “не ощущаются”, нет катарсиса.
- Развязка приходит “из воздуха”.
- Setup было много, payoff мало или наоборот.
- Нужно закрыть центральный вопрос истории.

## 3) WHEN NOT TO USE
- Ты строишь отдельную сцену конфликта (см. Conflict Dynamics).
- Ты пишешь сериал с намеренно открытым финалом (но даже там должен быть “локальный payoff”).

---

## 4) INPUTS (MINIMUM)
- Центральный конфликт и его ставки.
- Главный выбор героя (или его отсутствие как трагедия).
- Список setup/payoff (минимум 3).
- Ограничения мира (что нельзя нарушать без цены).

## 5) OUTPUTS (MINIMUM)
- Climax Card (решение/цена/результат).
- Payoff Closure List (что закрыли).
- Aftermath State (новое состояние).
- PASS/REWORK/FAIL.

---

## 6) DEFINITIONS (STRICT)
### Central question
Главный вопрос истории:
- “Сможет ли герой ___ ценой ___?”

### Decisive choice
Выбор, который нельзя отменить, и который определяет исход.

### Peak cost
Самая высокая цена истории (не обязательно смерть; может быть потеря себя/любви/дома/контроля).

### Resolution
Стабилизация после пика: мир новый, и у него новые правила.

### Payoff closure
Закрытие обещаний: подготовленное должно быть использовано/объяснено.

---

## 7) CLIMAX DESIGN PRINCIPLES
- P1: Кульминация = **решение**, а не “случайное событие”.
- P2: Цена в кульминации максимальна и видима.
- P3: Победа без цены = чит.
- P4: Развязка показывает **последствия**, а не только “результат”.
- P5: Setup/payoff закрывается (или намеренно оставляется с меткой).

---

## 8) PROCEDURE (CLIMAX CARD + PAYOFF AUDIT)
### Step 1 — Write CENTRAL QUESTION
Формат:
- “Сможет ли герой ___, не потеряв ___?”

### Step 2 — Define DECISIVE CHOICE (A vs B)
Минимум 2 опции:
- Option A:
- Option B:
и почему обе имеют цену.

### Step 3 — Define PEAK COST (what is paid)
Определи:
- кто платит
- чем платит
- что ломается/теряется необратимо

### Step 4 — Define CLIMAX ACTION (the decisive act)
Одна строка:
- “Герой делает ___, что приводит к ___.”

### Step 5 — Define OUTCOME (win/lose/pyrrhic)
- Win with cost
- Lose with meaning
- Pyrrhic win (победа ценой почти всего)

### Step 6 — Payoff Closure List (setup/payoff ledger)
Список 3–10 обещаний:
- Setup:
- Payoff:
- Status: CLOSED / OPEN (intentional) / MISSING

### Step 7 — Aftermath State (new stable world)
Заполни “AFTER state”:
- Goal changed:
- Stakes changed:
- Power changed:
- Access changed:
- Relationships changed:
- New constraints:

### Step 8 — Anti-cheat checks
Проверить:
- нет deus-ex инструмента без setup
- цена не “символическая”
- правила мира не нарушены без последствий
- финал отвечает на central question

## 10) QUICK CHECKLIST (PASS/FAIL)
- [ ] Есть central question
- [ ] Есть decisive choice (2 опции)
- [ ] Есть peak cost с необратимостью
- [ ] Есть outcome с последствиями
- [ ] Есть payoff closure list (3+)
- [ ] Есть aftermath state (что изменилось)
- [ ] Anti-cheat checks пройдены

PASS IF:
- все пункты true

REWORK IF:
- есть кульминация, но цена слабая или payoff не закрыт

FAIL IF:
- финал решает всё без выбора/цены/последствий

---

## 11) REWRITE TOOLKIT (FIX ENDINGS)
- Fix A: Replace “event climax” with “choice climax”
  Событие → решение героя.
- Fix B: Raise cost
  Увеличить цену (ресурс/отношение/идентичность).
- Fix C: Close setups
  Пройтись по setup/payoff и закрыть “висяки”.
- Fix D: Show aftermath
  Добавить 1–2 сцены последствий (новые правила).
- Fix E: Remove deus-ex
  Если решение “появилось” — добавить setup или удалить.

## 12) EXAMPLES (MANDATORY)
### 12.1 PASS EXAMPLE (pyrrhic win)
Central question:
- “Сможет ли герой спасти союзника, не потеряв свободу?”
Decisive choice:
- A: спасти союзника и раскрыться
- B: уйти и сохранить прикрытие
Peak cost:
- герой раскрыт, охота запускается (irreversible)
Payoff closure:
- “метка наблюдения” сработала → погоня (setup→payoff)
Aftermath:
- мир стал враждебнее, союзники меньше доверяют, цель меняется на выживание
WHY PASS:
- выбор, цена, закрытие обещаний, новое состояние.

### 12.2 FAIL EXAMPLE
В финале появляется “новое устройство”, которое решает всё без цены.
WHY FAIL:
- deus-ex + нет setup + нет cost.
