## PURPOSE

Символ — это объект/образ/ритуал, который:
- повторяется
- несёт смысл
- меняет смысл через контекст (эволюция)

Движок задаёт:
- карту символов
- правила использования
- “перегруз” (не превращать всё в символ)

---

## INPUTS

- Theme targets
- World belief/myth layer
- Scene needs
- Character values

---

## OUTPUTS

- Symbol map (symbol → meaning)
- Usage constraints
- Evolution rules (meaning shifts)
- Symbol placement plan (where/when)

---

## SYMBOL MAP (CANON)

For each symbol:
- SYMBOL (object/ritual/color/sound)
- CORE MEANING (1 line)
- ASSOCIATIONS (2–5)
- OPPOSITION (what it contradicts)
- APPEARANCE RULES (where allowed)
- EVOLUTION (how meaning can shift)
- TABOO/OVERUSE LIMIT

---

## PROCEDURE

1) Choose 1–3 core symbols per arc/realm
2) Tie each to theme + belief system
3) Define opposition (meaning contrast)
4) Define appearance rules and limits
5) Place symbols at turning points / climaxes only when needed
6) Track evolution across timeline

---

## VALIDATION RULES

- SY1: Symbol must have stable core meaning.
- SY2: Symbol must be anchored in world culture/belief.
- SY3: Overuse kills meaning → enforce limits.
- SY4: Meaning shifts require narrative cause.
