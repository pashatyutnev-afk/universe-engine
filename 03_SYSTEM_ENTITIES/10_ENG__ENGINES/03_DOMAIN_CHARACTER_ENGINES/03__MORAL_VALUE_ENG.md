---

## PURPOSE

Фиксирует **ценностную систему персонажа**:
- что “правильно/неправильно” для него
- где его границы
- за что он испытывает вину/стыд
- какие компромиссы он способен сделать
- где сломается

---

## INPUTS

- Character Core Card
- Cultural/world moral context (законы, религии, нормы)
- Key pressure situations (сцены выбора)

---

## OUTPUTS

- Value Stack (ценности в порядке важности)
- Moral Boundaries (таблица “можно/нельзя/под давлением”)
- Guilt/Shame triggers
- Moral dilemma templates for scenes

---

## CANONICAL VALUE STACK

- TOP VALUES (3–7) with ranking
- TABOOS (absolute)
- COMPROMISE ZONE (можно нарушить, но с ценой)
- JUSTIFICATION STYLE (как оправдывает)
- GUILT PROFILE (что вызывает вину)
- SHAME PROFILE (что вызывает стыд)
- REDEMPTION PATH (как он восстанавливает себя)

---

## PROCEDURE

1) Extract values from Value axis + background
2) Rank values by “what he sacrifices first”
3) Define taboos
4) Define compromise zone
5) Define guilt/shame mechanics
6) Build at least 2 dilemmas:
   - value vs value
   - value vs survival

---

## VALIDATION RULES

- V1: Ценности должны конфликтовать с историей.
- V2: Табу должны быть проверяемы (иначе фейк).
- V3: Компромисс должен иметь цену (психологическую/социальную).
- V4: Оправдания должны соответствовать психике (не “авторские”).

---

## FAILURE MODES

- ценности не проявляются в действиях
- “герой всегда правильный”
- нарушение границ без последствий

---

## INTEGRATION

- With THEME_MEANING_ENG: тема часто = ценностный конфликт
- With GROWTH_TRAUMA_ENG: вина/стыд → травма/рост
- With BEHAVIOR_ENG: ценности фильтруют решения
