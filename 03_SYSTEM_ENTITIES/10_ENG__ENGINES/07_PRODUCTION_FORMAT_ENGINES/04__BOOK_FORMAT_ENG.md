## PURPOSE

Книга — это:
- длинное погружение
- внутренние смыслы
- контроль ритма через главы/абзацы/переходы

Движок задаёт:
- структуру книги (acts/parts)
- тип главы (POV, цель, крючок)
- правила ритма
- правила экспозиции

---

## INPUTS

- Genre spec
- Narrative arc plan
- Character/world constraints
- Intended length

---

## OUTPUTS

- Book format spec
- Chapter template
- Act/part plan
- Pacing targets (slow/fast zones)

---

## BOOK FORMAT SPEC (CANON)

- POV strategy (1st/3rd/multi)
- Chapter length range (min/avg/max)
- Chapter purpose types:
  - Plot / Character / World / Reveal / Breath
- Hook rules (start of chapter)
- End rules (soft stop/cliffhanger)
- Exposition policy (drip/blocks)

---

## CHAPTER TEMPLATE (CANON)

- CHAPTER GOAL
- POV + distance
- KEY BEATS (3–7)
- TENSION MOVEMENT (up/down)
- NEW INFO (0–2 items)
- END STATE + HOOK

---

## VALIDATION RULES

- BF1: Каждая глава имеет цель и изменение состояния.
- BF2: Главы не заканчиваются “в никуда”.
- BF3: Экспозиция дозируется, не душит сюжет.
