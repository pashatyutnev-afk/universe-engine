ROLE: Models disruptive shocks that break assumptions and trigger cascading failures and adaptations

---

## PURPOSE

Системный шок — событие, которое:
- ломает базовые предположения системы
- вызывает каскадные отказы
- заставляет акторов адаптироваться
- меняет правила игры

Примеры:
- коллапс легитимности
- отключение инфраструктуры
- эпидемия
- технологический скачок/катастрофа
- “аномалия” (если мир допускает)

---

## INPUTS

- Systems map (power/resources/belief/tech/ecology)
- Current dependencies and chokepoints
- Timeline/epoch pressure

---

## OUTPUTS

- Shock Spec
- Cascade map (failures propagation)
- Adaptation options list
- New constraints / laws changes proposal

---

## SHOCK SPEC (CANON)

- SH_ID / NAME
- BROKEN ASSUMPTION (что считали стабильным)
- INITIAL IMPACT (0-hop)
- CASCADE (1–3 hops)
- PANIC/CONTROL DYNAMICS (как реагируют институты)
- ADAPTATION (новые практики)
- NEW STATUS QUO (если шок эпохальный)
- TRACE (что останется навсегда)

---

## PROCEDURE

1) Choose broken assumption
2) Identify primary impacted systems
3) Build cascade graph (with delays)
4) Add institutional response (control vs collapse)
5) Add adaptation innovations
6) Decide if epoch turn (timeline)

---

## VALIDATION RULES

- SS1: Шок должен ломать правило/ожидание, не просто “беда случилась”.
- SS2: Каскад обязателен (иначе не системный).
- SS3: Должна быть адаптация (люди не стоят на месте).
- SS4: След должен быть виден потом.
