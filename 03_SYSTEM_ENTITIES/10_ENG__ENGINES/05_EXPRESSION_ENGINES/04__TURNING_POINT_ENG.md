ROLE: Designs turning points that irreversibly reframe goals, stakes, or power distribution

---

## PURPOSE

Создаёт “повороты” как **необратимые изменения**:
- меняют цель
- меняют ставки
- меняют баланс сил
- раскрывают новую причинность

---

## INPUTS

- Conflict spec
- Causal chain
- Character revelations needs
- World constraints

---

## OUTPUTS

- Turning Point Spec
- Before/After state delta
- New options + closed options list

---

## TURNING POINT SPEC (CANON)

- TP_ID / NAME
- TYPE: reveal / betrayal / loss / discovery / inversion / escalation
- SETUP (что подводит)
- PAYOFF (что происходит)
- REVERSAL (что стало невозможным)
- NEW FRAME (новая реальность конфликта)
- COST (цена)
- TRACE (след)

---

## PROCEDURE

1) Identify what must change (goal/stakes/power)
2) Choose turning type
3) Write setup breadcrumbs (foreshadow)
4) Define payoff as event steps
5) Compute reversal (закрытые двери)
6) Compute new frame (новые правила)
7) Tie to causal chain

---

## VALIDATION RULES

- TP1: После поворота нельзя вернуться “как было”.
- TP2: У поворота должен быть setup (даже минимальный).
- TP3: Поворот должен менять набор вариантов действий.
- TP4: Цена есть всегда.
