---

## PURPOSE

Формирует кульминацию как:
- пик ставки
- пик цены
- пик решения (выбор)
- окончательное перераспределение сил

---

## INPUTS

- Conflict spec + escalation ladder
- Turning points
- Constraints (world law)

---

## OUTPUTS

- Climax Spec
- Decision node (choice tree)
- Cost ledger
- Post-climax state delta

---

## CLIMAX SPEC (CANON)

- CX_ID / NAME
- CONFLICT CORE (что решаем)
- ARENA (где и почему там)
- DECISION NODE (какой выбор обязателен)
- MAX COST (что будет потеряно)
- OUTCOME SET (2–4 возможных исхода)
- IRREVERSIBLE CHANGE (что меняется навсегда)
- TRACE (легенда/шрам/закон)

---

## PROCEDURE

1) Identify core question (win/lose/transform?)
2) Pick arena (логика chokepoint/символ)
3) Build decision node (choices + prices)
4) Ensure cost is maximum so far
5) Define outcomes with consequences
6) Lock irreversible change
7) Tie to resolution paths

---

## VALIDATION RULES

- CX1: Кульминация требует выбора, не просто “битва”.
- CX2: Цена выше предыдущих.
- CX3: Исход меняет систему (власть/мораль/ресурсы).
- CX4: Должен остаться след в каноне.
