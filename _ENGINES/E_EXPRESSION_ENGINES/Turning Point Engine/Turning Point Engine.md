# Turning Point Engine
## Движок Переломных Точек

---

## 1) Identity / Идентификация

- Engine ID: TPE-E4-004
- Class: Event Engine
- Level: E4 (Turning Point Core)
- Status: Active
- Owner: Universe Engine

---

## 2) Purpose / Назначение

### EN
Defines events that irreversibly change the direction, state, or trajectory of systems.
Controls how and when a sequence of events becomes a point of no return.

### RU
Определяет события, которые необратимо меняют направление, состояние или траекторию систем.
Управляет тем, как и когда цепочка событий становится точкой невозврата.

---

## 3) Responsibilities / Ответственность

Responsible for:
- Turning point classification
- Irreversibility rules
- State-shift validation
- Threshold crossing logic
- Pre- and post-point state enforcement

Отвечает за:
- Классификацию переломных точек
- Правила необратимости
- Валидацию смены состояния
- Логику пересечения порогов
- Принудительное различие состояний «до» и «после»

## 5) Inputs / Входные данные

Receives:
- Causal chains

- Accumulated effects
- World and system states
- Canon rules
- Temporal context

---

## 6) Outputs / Выходные данные

Produces:
- Turning point registrations
- Irreversibility markers
- State divergence signals
- Pre/Post state locks
- Validation reports

---

## 7) Operating Logic / Логика работы

1. Monitor accumulated causes and effects
2. Detect threshold crossings
3. Validate irreversibility conditions
4. Register turning point occurrence
5. Lock pre/post state separation
6. Publish turning point impact

---

## 8) Constraints / Ограничения

This engine must NOT:
- Create events independently
- Resolve conflicts
- Assign narrative meaning
- Override world laws
- Allow rollback without authority
