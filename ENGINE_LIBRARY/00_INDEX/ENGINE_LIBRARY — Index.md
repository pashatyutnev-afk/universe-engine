# ENGINE_LIBRARY — Index
## Канонический индекс библиотеки движков (L0)

---

## 0. Статус

- Layer: L0 (Meta / Standards)
- Scope: Engine Architecture & Documentation
- Authority: Governance Layer
- Status: Canonical
- Change Policy: Governance-controlled

---

## 1. Назначение

ENGINE_LIBRARY определяет **КАК** проектировать и документировать движки.

Она:
- задаёт стандарты
- даёт шаблоны
- фиксирует термины
- содержит чеклисты и тест-кейсы

Она НЕ:
- исполняет логику
- не принимает решения
- не хранит канон мира
- не заменяет ENGINES / SPECIALISTS / KNOWLEDGE_BASE

---

## 2. Связь с системой

- ENGINES → выполняют
- SPECIALISTS → формируют намерение
- KNOWLEDGE_BASE → информирует
- GOVERNANCE → разрешает/запрещает
- AUDIT LOG → фиксирует факты
- ENGINE_LIBRARY → задаёт стандарт построения всего выше

---

## 3. Обязательные разделы библиотеки

- 06_ENGINE_TEST_CASES
- 07_ENGINE_TERMS
- SYSTEM_WORK_INSTRUCTION

---

## 4. Абсолютные правила

1) Если термин не определён в 07_ENGINE_TERMS — его нельзя использовать в каноне.
2) Движок без паспорта/протокола/failure-модели не может стать FINAL.
3) ENGINE_LIBRARY описывает форму и границы, но не смысл и контент мира.

---

ENGINE_LIBRARY = правила создания правил.
