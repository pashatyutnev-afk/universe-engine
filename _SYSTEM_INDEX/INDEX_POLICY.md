# Index Policy
## Политика Индексов

Indexes are living maps of what actually exists in the vault.

Индексы — это живая карта того, что реально существует в хранилище.

Rule:
- Create entities first (folders/files).
- Update indexes after creation.
- Indexes contain navigation, not essays.

Правило:
- Сначала создаём сущности (папки/файлы).
- Потом обновляем индексы.
- Индексы — это навигация, а не сочинение.

Indexes are updated only when the assistant has read the current structure.

Индексы обновляются только после того, как ассистент прочитал текущую структуру.
