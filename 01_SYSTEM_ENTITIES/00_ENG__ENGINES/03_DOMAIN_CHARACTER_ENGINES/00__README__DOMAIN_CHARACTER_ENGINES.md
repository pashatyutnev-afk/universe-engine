## 1. PURPOSE

Domain Character Engines define **how characters are built, simulated, and evolved** inside the Universe Engine system.

They provide:
- character core model (identity + invariants)
- motivations and desires (drives)
- values and morality (decision filters)
- psychology (internal dynamics)
- behavior (observable outputs)
- relationships (network constraints)
- dialogue intent and speech naturalization
- growth/trauma modeling
- evolution over arcs

These engines output *character artifacts* used by narrative/world engines.

## 2. REALM CONTRACT (INPUT / OUTPUT)

### INPUT (what this realm accepts)
- world constraints (laws, culture, tech level)
- narrative constraints (arc goals, stakes)
- role in story (protagonist, antagonist, mentor, etc.)
- past events and state snapshots (from continuity)

### OUTPUT (what this realm produces)
- character profiles and passports (domain-level)
- motivation maps and value matrices
- behavior predictions (tendencies)
- relationship maps
- dialogue intent specs (not final prose)
- growth/evolution plans

Outputs are non-binding unless registered/approved by project governance.
