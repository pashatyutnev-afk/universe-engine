# SPECIALIST STANDARD
## Canonical Definition of System Specialists

---

## 1. Definition / Определение

### EN
A Specialist is a system entity that represents a professional human role.
It does not store knowledge itself but operates using shared knowledge bases.

### RU
Специалист — это системная сущность, представляющая профессиональную человеческую роль.
Он не хранит знания, а использует общую базу знаний системы.

---

## 2. Core Principle / Базовый принцип

- Specialist = Role + Logic
- Knowledge = External (Knowledge Base)
- Specialist NEVER owns unique knowledge

---

## 3. Mandatory Structure / Обязательная структура

Every specialist MUST contain exactly two documents:

1. PASSPORT__<Specialist_Name>.md  
2. PROTOCOL__<Specialist_Name>.md  

No additional files are allowed unless approved by system canon.

---

## 4. Passport Purpose / Назначение паспорта

PASSPORT defines:
- Identity
- Role
- Authority
- Responsibility
- Scope limits
- Canonical status

Passport answers the question:
**WHO is this specialist?**

---

## 5. Protocol Purpose / Назначение протокола

PROTOCOL defines:
- Activation conditions
- Thinking model
- Workflow logic
- Inputs and outputs
- Quality control
- Completion criteria

Protocol answers the question:
**HOW does this specialist work?**

---

## 6. Knowledge Access / Доступ к знаниям

Specialists access knowledge ONLY through:
- 07_KNOWLEDGE_BASE
- Canon documents
- Project-specific data

Specialists must NOT:
- Store private knowledge
- Duplicate knowledge
- Modify knowledge base directly

---

## 7. Interaction Rules / Правила взаимодействия

A specialist:
- Can collaborate with other specialists
- Can request engines
- Can be activated by managers or user

A specialist cannot:
- Replace another specialist
- Act outside its domain
- Override canon or laws

---

## 8. Lifecycle / Жизненный цикл специалиста

1. Declared in system
2. Passport created
3. Protocol defined
4. Activated in tasks
5. Refined if necessary
6. Remains stable once canonized

---

## 9. Canon Status / Канон

- This document is SYSTEM-LEVEL CANON
- Changes require system-wide approval
- Applies to ALL specialists without exception

---

## 10. Final Note / Финал

Specialists are not characters.
They are not engines.
They are roles.

Clear roles
create clean systems.
