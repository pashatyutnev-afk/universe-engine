# INDEX DOCUMENT — CANONICAL INSTRUCTION
# Единый шаблон INDEX-документа — Каноническая инструкция

---

## 1. Назначение INDEX-документа

### EN
The INDEX document defines the semantic identity, rules, and operating boundaries of a folder.
It is NOT a file list.  
It is a cognitive contract for AI and system logic.

### RU
INDEX-документ определяет смысловую идентичность, правила и границы папки.
Это НЕ список файлов.  
Это когнитивный контракт для ИИ и всей системы.

---

## 2. Ключевой принцип (ОБЯЗАТЕЛЬНО)

> INDEX describes **how to think**, not **what exists**.

Индекс:
- не перечисляет файлы
- не фиксирует финальное состояние
- не требует актуализации при каждом новом документе

Он задаёт **намерение, правила и структуру мышления**.

---

## 3. Когда создаётся INDEX

INDEX создаётся:
- сразу после создания папки
- ДО наполнения данными
- ДО появления файлов

❌ Запрещено создавать папку без INDEX  
❌ Запрещено наполнять папку без INDEX

---

## 4. Стандартное имя INDEX-файла

Обязательное имя:

Причины:
- читается первым
- приоритетен для ИИ
- единый стандарт во всей системе

---

## 5. Стандартная структура INDEX-документа

Каждый INDEX ОБЯЗАН содержать следующие блоки:

### 5.1 Identity / Идентификация
- Folder ID
- Domain
- Level (System / Engine / Specialist / Knowledge / Project)
- Status (Draft / Active / Frozen)

---

### 5.2 Purpose / Назначение
Отвечает на вопрос:
> ЗАЧЕМ СУЩЕСТВУЕТ ЭТА ПАПКА?

Коротко. Чётко. Без философии.

---

### 5.3 Scope / Область ответственности
Определяет:
- какие типы знаний здесь допустимы
- какие задачи папка обслуживает

---

### 5.4 Allowed Content / Разрешённый контент
Описываются ТИПЫ файлов, а не конкретные имена:

Пример:
- Concept definitions
- Analytical frameworks
- Patterns & heuristics
- Reference materials

---

### 5.5 Forbidden Content / Запрещённый контент
Критически важно.

Пример:
- Narrative events
- Character actions
- Project-specific canon
- Raw creative drafts

---

### 5.6 File Types / Типы документов
Описываются КЛАССЫ документов:

Пример:
- Knowledge Unit
- Pattern Description
- Rule Definition
- Reference Note

---

### 5.7 Usage Rules / Правила использования
- кто использует папку
- как ИИ должен к ней обращаться
- когда можно создавать новые файлы

---

### 5.8 Dependencies / Зависимости
Откуда папка может брать данные.

Пример:
- 01_NARRATIVE
- 02_PSYCHOLOGY_EMOTION

---

### 5.9 Used By / Используется
Кто использует данные этой папки:

- Engines
- Specialists
- Pipelines
- Projects

---

### 5.10 Evolution Rules / Правила эволюции
- можно ли расширять
- можно ли дробить
- что считается нарушением структуры

---

### 5.11 Canon Status / Канонический статус
- Canon Level
- Can be overridden? (Yes / No)
- Authority Engine (если есть)

---

## 6. Как INDEX используется ИИ

ИИ ОБЯЗАН:
1. Прочитать INDEX первым
2. Принять правила как абсолютные
3. Искать данные по СМЫСЛУ, а не по имени
4. Не создавать запрещённый контент
5. Соблюдать границы папки

---

## 7. Что INDEX НЕ делает

INDEX:
- ❌ не перечисляет все файлы
- ❌ не хранит знания
- ❌ не дублирует документацию
- ❌ не заменяет шаблоны

INDEX = навигационный интеллект.

---

## 8. Связь с другими INDEX

INDEX-документы:
- могут ссылаться друг на друга
- образуют смысловую карту системы
- не должны конфликтовать

При конфликте:
приоритет выше у:
System → Engine → Specialist → Knowledge → Project

## 9. Запреты (ЖЁСТКО)

Запрещено:

- смешивать разные домены в одном INDEX
- использовать INDEX как README
